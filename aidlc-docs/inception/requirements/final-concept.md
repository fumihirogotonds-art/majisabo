## ビジョン

> 「頑張らないことは、サボりではなく最適化である」
>
> 日本社会に根付く「勤勉＝美徳」の固定観念をデータに基づいて問い直し、
> 「正しいサボり方」を身につけることで、無駄な努力を手放せる人を増やす。
>
> このサービスは便利ツールではない。**サボり方の学習サービス**である。
> 使い続けるうちにサボり方を覚え、最終的にはサービスなしでもサボれるようになる。

## サービスの本質

**マジメニサボルは「効率化ツール」ではなく「サボり方を教えるサービス」。**

| 効率化ツール | マジメニサボル |
|------------|-------------|
| 「この作業を自動化します」 | 「この作業、やらなくていいです」 |
| 便利にする | **サボり方を覚えさせる** |
| ツールに依存し続ける | **卒業できる**（サボり方を身につけたら不要になる） |
| 生産性を上げる | **不要な生産性を捨てる** |

ユーザーは使い続けるうちに：
1. 「何が偽マジメか」を見分ける目が養われる
2. 「サボっても問題ない」という実績データで自信がつく
3. 最終的にサービスなしでも「正しいサボり方」ができるようになる

## サービス概要

あなたの日常には3種類の行動があります。

- **神サボリ** — やらなくても何も困らないこと（例：誰も反応しないチャンネルで毎日発言）
- **罪サボリ** — サボると後で困ること（例：自分宛のメンションを放置）
- **偽マジメ** — 頑張っているけど実は無意味なこと（例：18時以降にSlackで即レス）

このサービスは、チャット（Slack）のデータを分析し、
あなたの業務コミュニケーションをこの3つに分類します。

「ここはサボっていい」「ここはサボっちゃダメ」「ここは頑張る意味がない」を教えてくれる。

**そして、なぜそう判定したかを毎日説明する。**
ユーザーは判定理由を見て「サボりの判断基準」を学んでいく。

## 判定対象

| カテゴリ | 判定内容 | データソース | MVPスコープ |
|---------|---------|------------|:---:|
| **チャット（Slack）** | この発言に価値はあるか。即レスは必要か。このチャンネルに居る意味はあるか。時間外対応は偽マジメ | Slack API or エクスポート | **予選Must** |
| **会議（Slack経由）** | この会議に出る必要があるか。Slack上の会議招集・リマインドから検出 | Slackメッセージ内の会議情報（カレンダーBot通知、Zoom/Teamsリンク、招集メッセージ） | **予選Should** |
| **ファイル共有分析** | 共有したファイルは見られているか。ダウンロード0回＝偽マジメ | Slack API（files.info: ダウンロード数） | **予選Should** |
| **ステータス監視** | 定時後にオンラインのまま＝偽マジメ。有効/無効切り替え可能 | Slack API（users.getPresence） | **予選Should** |
| **DM効率化提案** | 同じ人とのDMが多すぎる場合、チャンネル化を提案＝神サボリ | Slack API（DM頻度分析） | **予選Should** |
| **メール** | この返信は必要か。このCC、読んでる人いるか | メール転送 or エクスポート | 予選Should |
| **Bot通知分析** | 一度もクリックしていないBot通知＝ミュート推奨 | Slack API（Bot投稿のリアクション/クリック） | 将来 |
| **タスク** | このタスクは本当に必要か。80%で十分では | 自己登録 or ツール連携 | 将来 |

### データ取得方式

**Slack連携（メイン）**:
- Slack App（Bot Token + Events API）でワークスペースにインストール
- チャンネル一覧、メッセージ履歴、リアクション数、メンション、スレッド返信を取得
- **本文もAI分析に使用**（質問・依頼・期限の有無を判定するため）
- 分析後、本文は保持しない（メタデータと判定結果のみ保存）
- フォールバック: エクスポートファイル（JSON/CSV）アップロード、テキストファイルインポート

**メール連携（Should）**:
- 自動転送方式 or エクスポートファイルアップロード
- 本文はAI分析後に破棄

## 判定ロジック（チャット判定フロー）

```
チャンネルの発言/メンション
    │
    ├─ 自分へのダイレクトメンション？
    │   ├─ Yes → 本文AI分析: 質問・依頼・期限あり？
    │   │         ├─ Yes → 🔴 罪サボリ（要対応）
    │   │         └─ No  → 🟢 神サボリ（既読OK）
    │   │
    │   └─ No（チャンネル全体の発言）
    │        │
    │        ├─ 過去30日で自分へのメンション回数は？
    │        │   ├─ 0回 → 🟢 神サボリ（ミュート推奨）
    │        │   │         「あなたはこのチャンネルに呼ばれていません」
    │        │   │
    │        │   └─ 1回以上 → 自分の発言への反応は？
    │        │                 ├─ 平均1件以上 → 通常参加
    │        │                 ├─ 平均0.5件以下（3回以上）→ 🟡 偽マジメ
    │        │                 │   「発言しても誰も反応しません」
    │        │                 └─ 反応なしだが情報共有目的？
    │        │                      └─ AI分析で判定 → 正常（報告系は反応不要）
    │        │
    │        └─ 時間外（定時後）の発言？
    │             └─ Yes → 🟡 偽マジメ（時間外対応は不要）
    │                       「この時間の対応は偽マジメです。明日の自分に任せましょう」
```

### 「反応なし＝偽マジメ」ではないパターン

| パターン | 判定 | 理由 |
|---------|------|------|
| 上司への報告 → 反応なし | **正常** | 報告は受領されれば十分。反応不要 |
| 情報共有の投稿 → 反応なし | **正常** | 読まれていれば価値あり（閲覧数で判定） |
| 質問への回答 → 反応なし | **正常** | 質問者が解決すれば十分 |
| 雑談チャンネルでの発言 → 反応なし | **偽マジメ候補** | 誰も反応しない雑談は時間の浪費 |
| 全体通知チャンネルでの発言 → 反応なし | **偽マジメ候補** | 求められていない発言 |

### 共通ルール
1. 偽マジメ判定は「3回以上同パターン」で確定（1回では判定しない）
2. 罪サボリ判定は「問題が発生した実績」がある場合のみ
3. 判定に迷う場合は「判定保留」（データ蓄積を待つ）
4. ユーザーが「問題あり」を報告したら即座に判定を修正
5. **判定理由を必ず説明する**（ユーザーがサボり方を「学ぶ」ため）

## サボりのPDCAサイクル（中核体験）

サービスの中核は「サボりのPDCA」を毎日回し、ユーザーにサボり方を教えること。

```
🎯 Plan（計画）— 事前判定
    │  「このチャンネル、あなたの発言価値: 低。ミュート推奨」
    │  「即レスは不要。平均4時間後でも問題なし」
    │  → 神サボリ候補を提示 + なぜサボっていいかの理由を説明
    │
    ▼
🚀 Do（実行）— 行動の記録
    │  → ユーザーが判定に従って行動（ミュート/既読スルー/時間外対応しない）
    │  → 行動結果を自動記録
    │
    ▼
📊 Check（評価）— 事後分析 + 学習
    │  「即レスしなかった → 影響: なし。サボって正解でした」
    │  「時間外に対応した → 翌朝でも問題なかったはず。次はサボりましょう」
    │  → ダメ度スコア算出
    │  → **なぜ不要だったかの理由を説明（学習ポイント）**
    │
    ▼
💡 Act（改善）— サボりスキルの向上
    │  → 「このチャンネルはミュートしても大丈夫だと学びました」
    │  → 「18時以降の対応は不要だと分かりました」
    │  → 次のPlanの判定精度が上がる
    │  → **ユーザー自身のサボり判断力も上がる**
    │
    └─→ 最終目標: サービスなしでもサボれるようになる
```

```
    ・メンションなしの投稿3件をスルー → 問題なし
    💡 学び: 「メンションがない投稿は、あなた宛ではありません」

  🟡 偽マジメ（無駄に頑張りました）: 2件
    ・誰も反応しないチャンネルで発言（10分の浪費）

    💡 学び: 「18時以降の対応は、ほぼ全て翌朝で間に合います」

  🔴 罪サボリ（サボっちゃダメでした）: 0件
    ・今日はパーフェクト！

  ⏰ 時間外労働アラート

    ・「この時間の対応は全て偽マジメです。
       明日の自分に任せましょう。」

  💰 今日の創出時間: 45分
  📈 今週の累計: 3.2時間
  🎓 サボりスキルLv: 初級 → 中級まであと3日

  💬 マジサボからの一言:

   それは勤勉ではなく偽マジメです。

```

## サボりレベル＆ロードマップ（学習体験の核心）

ユーザーが「判定結果を見るだけ」ではなく「段階的にサボり方を身につける」ための仕組み。

### サボりレベル（段階的ステップ）

| Lv | 名前 | チャレンジ内容 | 達成条件 |
|:---:|------|-------------|---------|
| 0 | サボリ入門 | 「了解しました」を👍リアクションに変える | 3日連続で定型返信をリアクションに置き換え |
| 1 | サボり見習い | 時間外のSlackを閉じる | 3日連続で18時以降の発言ゼロ |
| 2 | 初級サボリスト | メンションなしチャンネルをミュート | 1チャンネルミュートして1週間問題なし |
| 3 | 中級サボリスト | 即レスをやめる（2時間後でOK） | 即レス率50%以下を1週間維持 |
| 4 | 上級サボリスト | 偽マジメタスクを「やらない」と決断 | 偽マジメ判定されたアクションを1つスキップ |
| 5 | マスターサボリスト | チームに「この会議不要では？」と提案 | 改善提案を1つ実行 |

### サボりロードマップ（ダッシュボード表示）

```
  🗺️ あなたのサボりロードマップ

  ✅ Step 0: 「了解しました」を👍に変えた（達成！3日連続）
  ✅ Step 1: 時間外Slackをやめた（達成！3日連続）
  ✅ Step 2: #general をミュートした（達成！1週間問題なし）
  → Step 3: 即レスを2時間後に変える（挑戦中: 現在即レス率68%）
  ○ Step 4: 偽マジメタスクを1つ廃止する
  ○ Step 5: チームに改善提案する

  📊 現在のサボりLv: 3（中級サボリスト）
  🎯 次の目標: 即レス率を50%以下にする
```

### 明日のサボりチャレンジ（定時レスポンスに含める）

毎日の定時レスポンスに「明日やること」を1つ提示：

```
  💡 明日のサボりチャレンジ:
  「#project-updates を1日ミュートしてみましょう。
   メンションがなければ、あなたには不要なチャンネルです」

  難易度: ★☆☆☆☆（初心者向け）
  成功すると: サボりLv.2に近づきます
```

チャレンジはユーザーの現在のレベルに合わせて自動選択される。

## 設計方針（確定済み）

| 項目 | 決定 | 理由 |
|------|------|------|
| サービスの本質 | **サボり方の学習サービス**（便利ツールではない） | ユーザーがサボり方を身につけて「卒業」できることが目標 |
| 予選MVPのメインデータソース | **チャット（Slack）** | 即日PDCAが回る。メールは Should |
| Larry Wallの三大美徳 | **内部思想のみ** | UIには「神サボリ/罪サボリ/偽マジメ」の3分類のみ表示 |
| ゲーミフィケーション | **MVPには含めない** | 将来ビジョンとして残す |
| PDCAサイクル | **前面に出す** | 中核体験。毎日回してサボり方を学ぶ |
| 判定理由の説明 | **必須** | ユーザーが「なぜサボっていいか」を学ぶため |
| 本文のAI分析 | **する** | 質問・依頼・期限の判定に必要。分析後に本文は破棄 |
| 反応なし＝偽マジメではない | **パターン別に判定** | 報告系・情報共有系は反応なしでも正常 |
| チャンネル選択 | **初回セットアップ時にUI提供** | ユーザーが分析対象チャンネルを選択 |
| 行動追跡 | **48hリマインド + 24hメンションチェック × 7日** | 1週間メンション0回で「サボり成功」確定 |
| 一言コメント | **デモ: コメントプールから選択 / 本番: Bedrock生成** | デモの品質を担保 |

## ビジネスモデル（将来ビジョン）

| プラン | レスポンス頻度 | 内容 |
|--------|:---:|------|
| **無料プラン** | 1日1回（18:00） | 定時レスポンスのみ。基本的な判定＋ダメ度スコア |
| **有料プラン** | リアルタイム | 即座に判定結果が届く。詳細分析＋改善提案 |
| **卒業後（マスター）** | — | サボりを極める。MBO目標連携、チーム指導、組織改革 |

※ MVPでは無料プラン相当（1日1回）で実装。有料プランは将来ビジョン。

### 卒業後のユーザー像
- サボりLv.5（マスター）に到達したユーザーは「卒業」
- 卒業後は「サボりを極める」フェーズに移行：
  - MBOの目標として「偽マジメ削減」を設定
  - チームメンバーにサボり方を教える側になる
  - 組織の偽マジメコスト削減に貢献する
- **ビジネス的には**: 個人→チーム版へのアップグレード需要が生まれる

## データ保持方針

| 方針 | 内容 |
|------|------|
| **Unit単位でデータ保持** | 各Unitが自分のDynamoDBテーブルを持つ（マイクロサービス原則） |
| **共有可能なデータ** | マスタデータ、設定値など読み取り専用のデータのみ共有可 |
| **Unit間通信** | データが必要な場合はAPI経由 or EventBridgeイベントで取得（直接テーブル参照は禁止） |
| **本文データ** | AI分析後に破棄。メタデータと判定結果のみ保持 |
