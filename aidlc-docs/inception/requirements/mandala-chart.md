# 「人をダメにする」マンダラチャート

> **この分析の結論**: 64アイデアを俯瞰した結果、最も強いのは「**チャットの要否判定**」。
> メンションなしチャンネルのミュート推奨、即レス不要の判定、時間外対応の偽マジメ検出 — 
> これらが最終コンセプト「マジメニサボル（チャット特化のサボり学習サービス）」に集約された。

中心テーマ「人をダメにする」から8つのサブテーマに展開し、
さらに各サブテーマから8つの具体アイデアに展開します。

---

## マンダラチャート全体像

```
┌────────────┬────────────┬────────────┐
│            │            │            │
│  動かない  │  考えない  │  決めない  │
│            │            │            │
├────────────┼────────────┼────────────┤
│            │   人を     │            │
│  待たない  │  ダメに    │  頑張らない│
│            │   する     │            │
├────────────┼────────────┼────────────┤
│            │            │            │
│  関わらない│  覚えない  │  責任取らない│
│            │            │            │
└────────────┴────────────┴────────────┘
```

8つのサブテーマ:
1. **動かない** — 身体的行動を減らす
2. **考えない** — 認知的負荷を減らす
3. **決めない** — 意思決定を減らす
4. **頑張らない** — 意志力の消費を減らす
5. **待たない** — 忍耐を不要にする
6. **関わらない** — 社会的コストを減らす
7. **覚えない** — 記憶の負荷を減らす
8. **責任取らない** — 責任の重荷を減らす

## マンダラチャートの統合的発見

### パターン1: 「判定系」が最も新規性が高い

マンダラチャート全体を見ると、**「〇〇の要否を自動判定する」**系のアイデアが最も新規性が高い。
- 会議の要否判定
- タスクの要否判定
- 報告の要否判定
- 承認の要否判定
- 飲み会の要否判定

→ これは先ほどの「いいサボり / 悪いサボり判定」と完全に一致する。

### パターン2: 「限界点の算出」が実用性が高い

- 残業の限界効用ゼロ点
- 完璧主義の限界点（80%で十分）
- 会議の生産性限界（20分で急落）
- 気遣いの限界効用

→ 「ここまでやれば十分。これ以上は無意味」を科学的に示す。

### パターン3: 「責任転嫁」がユーモア的に最強

- AIに判断を委ねる
- 「AIが選びました」免責
- 「最善を尽くした」証明書

→ デモ映え・笑いのポイントとして使える。ただし単体ではサービスにならない。

## 結論: マンダラチャートが示す最適コンセプト

8つのサブテーマ × 8つのアイデア = 64アイデアを俯瞰した結果、
最も強いのは**「判定系 + 限界点算出 + ユーモア」の組み合わせ**。

これは案1+2の方向性（三大美徳 × 科学）と完全に合致し、
さらに「いいサボり / 悪いサボり」の判定軸で統合できる。

**マンダラチャートが裏付けたコンセプト:**
> カレンダー・タスクデータから「やらなくていいこと」を自動判定し、
> 科学的根拠とユーモアで「賢いサボり方」を提案するサービス。
