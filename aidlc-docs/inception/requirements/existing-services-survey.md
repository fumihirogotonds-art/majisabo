# 既存サービス調査（競合分析）

> **目的**: マジメニサボルの独自性を検証するため、類似サービスを調査し差別化ポイントを明確にする

---

## 調査対象カテゴリ

1. チャット分析ツール
2. 働き方分析ツール
3. 生産性向上ツール
4. サボり・怠惰系サービス

---

## 1. チャット分析ツール

### Slack Analytics（Slack公式）

| 項目 | 内容 |
|------|------|
| 概要 | Slackワークスペースの利用統計（メッセージ数、アクティブユーザー数等） |
| ターゲット | ワークスペース管理者 |
| 提供価値 | 「チームがSlackをどう使っているか」の可視化 |
| **マジサボとの違い** | 管理者向けの統計ツール。個人が「自分はサボっていいか」を判断する機能はない。「もっと使え」方向 |

### Swit / Loom Analytics

| 項目 | 内容 |
|------|------|
| 概要 | チームコミュニケーションの効率分析 |
| ターゲット | チームリーダー |
| 提供価値 | コミュニケーションのボトルネック特定 |
| **マジサボとの違い** | 「効率を上げる」目的。マジサボは「やらなくていいことを教える」目的。方向が真逆 |

---

## 2. 働き方分析ツール

### Microsoft Viva Insights

| 項目 | 内容 |
|------|------|
| 概要 | Microsoft 365のデータから働き方パターンを分析。会議時間、集中時間、コラボレーション時間を可視化 |
| ターゲット | Microsoft 365ユーザー（個人＋管理者） |
| 提供価値 | 「働きすぎ」の検出、集中時間の確保提案 |
| **マジサボとの違い** | 「バランスを取りましょう」という中立的な提案。マジサボは「サボれ」と明確に言う。3分類（神怠惰/罪怠惰/偽勤勉）のフレームワークがない。「サボった結果の追跡」機能がない |

### RescueTime

| 項目 | 内容 |
|------|------|
| 概要 | PC/スマホの利用時間を自動記録し、生産性スコアを算出 |
| ターゲット | 個人（フリーランス、リモートワーカー） |
| 提供価値 | 「何に時間を使ったか」の可視化 |
| **マジサボとの違い** | 「生産的な時間を増やせ」方向。マジサボは「不要な時間を減らせ」方向。アプリ利用時間の記録であり、コミュニケーションの要否判定はしない |

### Clockwise

| 項目 | 内容 |
|------|------|
| 概要 | カレンダーを最適化し、集中時間を自動確保 |
| ターゲット | カレンダーが会議で埋まっている人 |
| 提供価値 | 会議の自動リスケジュール、集中時間ブロック |
| **マジサボとの違い** | 「会議の時間を調整する」ツール。マジサボは「そもそも出るな」と言う。チャット分析機能はない |

## 3. 生産性向上ツール

### SaneBox

| 項目 | 内容 |
|------|------|
| 概要 | メールの優先度を自動分類（重要/後で/不要） |
| ターゲット | メールが多いビジネスパーソン |
| 提供価値 | 重要なメールだけに集中できる |
| **マジサボとの違い** | 「読むべきか」の判定。マジサボは「返信すべきか」「そもそも対応すべきか」の判定。「サボった結果の追跡」がない。学習サービスではない |

### Superhuman / Spark

| 項目 | 内容 |
|------|------|
| 概要 | 高速メールクライアント。AI要約、優先度表示 |
| ターゲット | メールヘビーユーザー |
| 提供価値 | メール処理の高速化 |
| **マジサボとの違い** | 「メールを速く処理する」ツール。マジサボは「メールを処理しなくていい」と教える。根本的に目的が異なる |

---

## 4. サボり・怠惰系サービス

### WeAreLazy（コンセプト段階）

| 項目 | 内容 |
|------|------|
| 概要 | 「怠惰を計測する」コンセプトのサービス（実サービスとしては確認できず） |
| **マジサボとの違い** | 計測のみ。判定・学習・行動変容の仕組みがない |

### 各種「言い訳ジェネレーター」

| 項目 | 内容 |
|------|------|
| 概要 | サボる理由を自動生成するジョークアプリ |
| **マジサボとの違い** | 完全なジョークアプリ。実用性ゼロ。マジサボは「半分ジョーク半分実用」で、実際にサボり方を学べる |

## 独自性の結論

**マジメニサボルは以下の点で既存サービスと完全に差別化される：**

1. **目的が真逆**: 既存ツールは「もっと頑張れ/効率化しろ」。マジサボは「もっとサボれ」
2. **3分類フレームワーク**: 神怠惰/罪怠惰/偽勤勉の分類は他に存在しない
3. **サボった結果の追跡**: 「やらなくても問題なかった」の実績データ蓄積は未開拓
4. **学習サービス**: 便利ツールではなく「サボり方を教えて卒業させる」コンセプトは唯一
5. **判定理由の説明**: なぜサボっていいかを毎日教える仕組みは他にない
6. **段階的レベルアップ**: サボりスキルの成長を可視化するサービスは存在しない

**直接的な競合は存在しない。**
