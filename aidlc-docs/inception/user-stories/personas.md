# ペルソナ定義

---

## ペルソナ1: 佐藤 健太（個人ユーザー）

> **MVPスコープ: 予選**

| 項目 | 内容 |
|------|------|
| 名前 | 佐藤 健太（さとう けんた） |
| 年齢 | 32歳 |
| 職種 | SIer のプロジェクトリーダー |
| チーム規模 | 8名のチームを率いる |
| 技術レベル | 中程度（Webアプリは日常的に使う） |

### 日常の課題
- Slackのチャンネルが20以上あり、全部チェックしている
- 「念のため対応」しているチャットが半分以上
- 即レスした後に「これ、明日でよかったな」と感じる
- チャット対応が多すぎて、本来の設計・レビュー業務が残業に回る
- サボりたいが「対応しないと何か言われるかも」という不安がある

### マジメニサボルに期待すること
- 「対応しなくていいチャット」をデータで教えてほしい
- サボっても問題なかった実績を見て安心したい
- 無意味な頑張り（偽マジメ）に気づきたい
- 創出した時間で本来の業務に集中したい

### 利用シーン
- 18:00: 定時レスポンスで「今日のサボり成績」を確認
- 翌朝: 「明日のサボりチャレンジ」に従って行動
- 週末: ダメ度トレンドを見て成長を実感

### 成功指標
- 偽マジメチャンネルを5つミュート
- 時間外のSlack対応をゼロに
- 「サボっても問題なかった」実績が20件以上蓄積

## ペルソナ2: 山下 翔太（若手社員）

> **MVPスコープ: 予選**

| 項目 | 内容 |
|------|------|
| 名前 | 山下 翔太（やました しょうた） |
| 年齢 | 25歳 |
| 職種 | SIer のシステムエンジニア（入社3年目） |
| チーム規模 | 12名のチームの一般メンバー |
| 技術レベル | 中程度（日常的にWebアプリを使う） |

### 日常の課題
- 先輩や上司が参加しているチャンネルで「新人だから全部対応しなきゃ」と思っている
- チャンネルの通知が多すぎて、自分の開発タスクが進まない
- 「対応しなくていいですか？」と聞く勇気がない
- 断ると「やる気がない」と思われそうで怖い
- 即レスしないと評価が下がると思い込んでいる

### マジメニサボルに期待すること
- 「あなたはこのチャンネルに居る必要がありません」とデータで言ってほしい
- 上司に見せられる「対応不要の根拠」がほしい
- 対応しなくても問題なかった実績を積み上げて、自信を持ちたい
- 自分の時間を開発に使いたい

### 利用シーン
- 18:00: 定時レスポンスで「今日の偽マジメ」を確認し、明日の行動を決める
- 翌朝: サボりチャレンジに従ってチャンネルをミュート
- 1on1: 上司に「マジサボのデータでは対応不要と出ています」と伝える

### 成功指標
- 「対応不要」のチャンネルを週3つ以上ミュートできるようになる
- 上司に断る際の心理的ハードルが下がる
- 開発に使える時間が週5時間以上増える

## ペルソナ3: 鈴木 あゆみ（リモートワーカー）

> **MVPスコープ: 予選**

| 項目 | 内容 |
|------|------|
| 名前 | 鈴木 あゆみ（すずき あゆみ） |
| 年齢 | 34歳 |
| 職種 | Webマーケティング会社のディレクター |
| チーム規模 | 6名のチーム（全員フルリモート） |
| 技術レベル | 中〜高（SaaSツールを多数活用） |

### 日常の課題
- フルリモートになってからSlackの通知が1.5倍に増えた（「ちょっと聞きたい」が全部チャットになる）
- 1日に参加しているチャンネルが20以上
- 即レスしないと「仕事してない」と思われる不安
- チャットの合間が15分しかなく、集中作業の時間が取れない
- 「このチャンネル、自分に関係ある？」と毎日思う

### マジメニサボルに期待すること
- 「自分に関係ないチャンネル」と「本当に見るべきチャンネル」を分けてほしい
- 即レスしなくても貢献していることをデータで証明したい
- チャンネルを減らして集中作業のブロック時間を確保したい
- チームに「このチャンネルは非同期でいいのでは」と提案する根拠がほしい

### 利用シーン
- 18:00: 定時レスポンスで「今日の偽マジメチャンネル」を確認
- 翌朝: サボりチャレンジに従ってチャンネルをミュート
- 週末: 「今週ミュートして問題なかったチャンネル」の数を確認

### 成功指標
- 監視チャンネル数が20→10に減少
- 集中作業のブロック時間が1日2時間以上確保できる
- 「ミュートしても問題なかった」実績が10件以上蓄積

## ペルソナ4: 田中 美咲（チームリーダー）

> **MVPスコープ: 決勝**

| 項目 | 内容 |
|------|------|
| 名前 | 田中 美咲（たなか みさき） |
| 年齢 | 38歳 |
| 職種 | 事業部の課長 |
| チーム規模 | 20名の部下を持つ |
| 技術レベル | 低〜中（ツールは使えるが設定は苦手） |

### 日常の課題
- 部下が「無意味なチャンネル監視」に時間を取られているのを知っている
- 偽マジメチャンネルを減らしたいが「廃止の根拠」がない
- 部門の生産性を上げたいが、何が無駄かの可視化ができていない
- 「偽マジメ」を指摘すると角が立つ

### マジメニサボルに期待すること
- チーム全体の「偽マジメコスト」を数値で見たい
- 「このチャンネルは統合しても問題ない」というデータが欲しい
- 部下に「サボっていいよ」と言える根拠が欲しい
- 組織改善の成果を上層部に報告したい

### 利用シーン
- 月初: チームの先月の偽マジメコストレポートを確認
- チャンネル見直し時: 「メンバーの貢献度データ」を根拠にチャンネルを整理
- 評価面談: 部下の「時間創出量」を評価指標として活用

### 成功指標
- チーム全体の偽マジメチャンネルを月5つ削減
- 偽マジメとして廃止・統合したチャンネルが3件以上
- 部下の時間外Slack対応が平均50%減少

## ペルソナ5: 中村 裕子（バックオフィス・人事総務）

> **MVPスコープ: 決勝**

| 項目 | 内容 |
|------|------|
| 名前 | 中村 裕子（なかむら ゆうこ） |
| 年齢 | 41歳 |
| 職種 | 人事総務部 マネージャー |
| 担当範囲 | 全社300名の働き方改革推進 |
| 技術レベル | 低（Excelとメールが中心） |

### 日常の課題
- 「働き方改革」を推進する立場だが、具体的な数値がない
- 「チャットが多すぎる」「即レス文化がきつい」という声は聞くが、実態を定量化できていない
- 施策を打っても効果測定ができない
- 経営層に「改革の成果」を報告する必要があるが、説得力のあるデータがない
- 各部門のコミュニケーション文化が異なり、全社一律の施策が難しい

### マジメニサボルに期待すること
- 全社のコミュニケーションコスト（時間×人件費）を可視化したい
- 部門別の「偽マジメ率」を比較したい
- 施策の前後で数値がどう変わったかを追跡したい
- 経営会議で使えるレポートを自動生成してほしい

### 利用シーン
- 四半期: 全社の偽マジメコストレポートを経営会議に提出
- 施策実施後: 「偽マジメ削減キャンペーン」の効果を数値で確認
- 年度計画: 来期の働き方改革KPIを設定する際の基礎データとして活用

### 成功指標
- 全社の偽マジメコストを四半期で10%削減
- 経営層が「コミュニケーション改革」の予算を承認する根拠データを提供
- 部門別の改善ランキングで健全な競争が生まれる
