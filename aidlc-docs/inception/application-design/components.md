# コンポーネント定義

---

## コンポーネント一覧

| # | コンポーネント | 責務 | 技術 |
|---|--------------|------|------|
| C1 | Auth | ユーザー認証・セッション管理 | Cognito |
| C2 | SlackConnector | Slackデータ取得・正規化 | Lambda + Slack API |
| C3 | JudgmentEngine | 神サボリ/罪サボリ/偽マジメの判定 | Lambda + Bedrock |
| C4 | ScoreCalculator | ダメ度スコア算出 | Lambda |
| C5 | DailyReporter | 定時レスポンス（18:00日次レポート）生成・配信 | EventBridge + Lambda + Slack Webhook |
| C6 | ActionTracker | 行動記録・結果追跡 | Lambda + DynamoDB |
| C7 | Dashboard API | フロントエンド向けデータ提供 | API Gateway + Lambda |
| C8 | Dashboard UI | ダッシュボード画面 | React (S3 + CloudFront) |

---

## コンポーネント詳細

### C1: Auth（認証）

| 項目 | 内容 |
|------|------|
| **責務** | ユーザー登録、ログイン、セッション管理、トークン発行 |
| **技術** | Amazon Cognito User Pool |
| **インターフェース** | サインアップ、サインイン、トークン検証 |
| **データ** | ユーザーID、メールアドレス、Slack連携状態 |

---

### C2: SlackConnector（Slackデータ取得）

| 項目 | 内容 |
|------|------|
| **責務** | Slackワークスペースからデータを取得し、正規化して保存。エクスポートファイルのパースも担当。初回セットアップ時に分析対象チャンネルを選択するUIを提供 |
| **技術** | Lambda + Slack Bot Token (conversations.history, conversations.list, users.info) |
| **インターフェース** | チャンネル一覧取得、分析対象チャンネル選択、メッセージ履歴取得、メンションチェック、エクスポートファイルパース |
| **データ** | チャンネルID、メッセージ（本文・タイムスタンプ・リアクション数・スレッド返信数・メンション） |
| **動作モード** | リアルタイム（Events API）+ バッチ（日次集計）+ メンションチェック（24時間ごと） |
| **初回セットアップ** | ユーザーが分析対象チャンネルを選択。選択されたチャンネルのみ分析対象 |

---

### C3: JudgmentEngine（判定エンジン）

| 項目 | 内容 |
|------|------|
| **責務** | チャットデータを分析し、神サボリ/罪サボリ/偽マジメに分類。判定理由を生成 |
| **技術** | Lambda + Amazon Bedrock (Claude) |
| **インターフェース** | メッセージ判定（リアルタイム簡易）、チャンネル判定（バッチ詳細）、判定理由生成 |
| **判定ロジック** | メタデータルール（メンション数、リアクション数、時間帯）+ AI本文分析（質問/依頼/期限の有無、報告系判定） |
| **動作モード** | 簡易判定: メタデータのみ（リアルタイム）/ 詳細判定: 本文AI分析含む（バッチ） |

---

### C4: ScoreCalculator（スコア算出）

| 項目 | 内容 |
|------|------|
| **責務** | 判定結果を集計し、ダメ度スコア（0〜100）を算出。トレンドデータを管理 |
| **技術** | Lambda |
| **インターフェース** | 日次スコア算出、週次トレンド算出、累計創出時間算出 |
| **算出式** | `(神サボリ件数 / 全判定件数) × 100 - (罪サボリ件数 × 10)` |
| **データ** | 日次スコア履歴、累計神サボリ/罪サボリ/偽マジメ件数、創出時間 |

---

### C5: DailyReporter（定時レスポンス）

| 項目 | 内容 |
|------|------|
| **責務** | 毎日18:00に日次レポートを生成し、Slack DMで配信。48時間リマインドも担当 |
| **技術** | EventBridge（スケジュール）+ Lambda + Slack Webhook |
| **インターフェース** | レポート生成、Slack DM送信、リマインド送信 |
| **レポート内容** | ダメ度スコア、神サボリ/偽マジメ/罪サボリの件数と具体例、学びポイント、時間外アラート、一言コメント |
| **一言コメント** | デモ用: 事前用意したコメントプールから選択。本番: Bedrock生成 |
| **ビジネスモデル** | 無料プラン: 1日1回（18:00）のレスポンス / 有料プラン: リアルタイムレスポンス |

---

### C6: ActionTracker（行動記録）

| 項目 | 内容 |
|------|------|
| **責務** | ユーザーの行動（ミュート実行、即レスしなかった等）を記録。メンションチェックによる自動追跡。問題発生有無の判定 |
| **技術** | Lambda + DynamoDB + EventBridge（24時間ごとのメンションチェック） |
| **インターフェース** | 行動記録、問題報告、メンションチェック、実績集計 |
| **データ** | 行動ID、行動タイプ（ミュート/スキップ/時間外不対応）、結果（問題なし/問題あり）、タイムスタンプ、追跡期間 |
| **追跡ロジック** | 48時間後にリマインド + 24時間ごとのメンションチェックを1週間実施 → メンション0回で「サボり成功」確定 |

---

### C7: Dashboard API（API層）

| 項目 | 内容 |
|------|------|
| **責務** | フロントエンドにデータを提供するREST API |
| **技術** | API Gateway + Lambda |
| **インターフェース** | ダッシュボードデータ取得、判定一覧取得、トレンドデータ取得、行動記録API |
| **認証** | Cognito Authorizer |

---

### C8: Dashboard UI（フロントエンド）

| 項目 | 内容 |
|------|------|
| **責務** | ダッシュボード画面の表示。ダメ度スコア、判定結果、トレンドグラフ、行動記録UI |
| **技術** | React + Vite（S3 + CloudFront配信） |
| **画面** | ダッシュボード（ダメ度＋サマリー）、Plan（判定一覧）、履歴（トレンドグラフ）、設定（Slack連携） |
